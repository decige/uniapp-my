# ccyuniapp — 个人在线简历（面试链接即用）

> 技术栈：Vue3 + UniApp + Node.js  
> 运行环境：HBuilderX（HX）多端适配，**主战场是 H5 网页**  
> 项目路径：`D:\vue\ccyuniapp`  
> 你的域名：面试时直接丢链接，不用带纸质/PDF 简历

---

## 一、项目概述

### 1.1 你是谁、要解决什么

**你是面试者**，不是 HR 系统管理员。

平时把 PDF 简历上传到后台，系统自动解析文字并排版成网页。面试时对方问「带简历了吗」，你直接说：

> 「我简历在这：`https://你的域名.com`」

对方用手机/电脑打开链接，立刻看到排版好的简历；需要的话还能点「查看原 PDF」。

### 1.2 典型使用场景

```
面试前（你自己）          面试中（面试官）              面试后（你自己）
──────────────          ──────────────              ──────────────
上传/更新 PDF 简历   →    打开你分享的域名链接     →    改 PDF 重新上传即可
后台解析 + 排版           看网页版简历                  链接不变，内容自动更新
```

### 1.3 核心功能

| 功能 | 谁用 | 说明 |
|------|------|------|
| 简历展示页（首页） | 面试官 | 打开域名直接看，无需登录，排版清晰 |
| 原 PDF 预览/下载 | 面试官 | 「查看原版」按钮，打开你上传的那份 PDF |
| 简历管理页（后台） | 你自己 | 上传/替换 PDF，触发重新解析（需简单鉴权） |
| 链接分享 | 你自己 | 固定域名，内容更新后链接不用改 |

### 1.4 页面角色划分

| 页面 | 路径示例 | 访问者 | 是否公开 |
|------|----------|--------|----------|
| 简历展示 | `/` 或 `/resume` | 面试官 | ✅ 公开 |
| 原 PDF 预览 | `/preview` | 面试官 | ✅ 公开 |
| 管理后台 | `/admin` | 你自己 | 🔒 需密码 |

> **重点**：面试官只会看到展示页，上传和管理是你私用的。

### 1.5 多端策略（面试场景优先 H5）

| 端 | 优先级 | 用途 |
|----|--------|------|
| **H5 网页** | ⭐⭐⭐ 最高 | 绑域名，面试分享链接，面试官手机/电脑都能开 |
| 微信小程序 | ⭐ 可选 | 自己方便管理简历，或做成「我的简历」小程序码 |
| App | ⭐ 低 | 一般不需要，H5 足够 |

## 三、后端（Node.js）实施步骤

> 后端逻辑很简单：**只维护一份「当前简历」**，不是多用户系统。

### 阶段 B1：环境搭建

- [ ] **B1.1** 初始化项目
  ```bash
  cd D:\vue\ccyuniapp\server
  npm init -y
  ```
- [ ] **B1.2** 安装依赖
  ```bash
  npm install express cors multer pdf-parse dotenv
  npm install -D nodemon
  ```
- [ ] **B1.3** 创建 `.env`
  ```
  PORT=3000
  UPLOAD_DIR=./uploads
  DATA_FILE=./data/resume.json
  ADMIN_PASSWORD=你的管理密码
  MAX_FILE_SIZE=10485760
  ```
- [ ] **B1.4** 配置启动脚本 `npm run dev` / `npm start`

### 阶段 B2：单份简历存储模型

- [ ] **B2.1** 固定存储路径：`uploads/resume.pdf`（新上传直接覆盖旧文件）
- [ ] **B2.2** 解析结果存 `data/resume.json`
  ```json
  {
    "rawText": "PDF 提取的全文",
    "sections": [
      { "title": "基本信息", "content": "姓名 / 手机 / 邮箱" },
      { "title": "工作经历", "items": ["公司A 2022-2024 ...", "..."] },
      { "title": "项目经历", "items": ["..."] },
      { "title": "技能", "items": ["Vue3", "Node.js"] }
    ],
    "updatedAt": "2026-06-29T10:00:00.000Z",
    "fileName": "张三_前端工程师.pdf"
  }
  ```
- [ ] **B2.3** 上传新 PDF → 覆盖文件 → 重新解析 → 更新 JSON

### 阶段 B3：PDF 解析模块

- [ ] **B3.1** 用 `pdf-parse` 提取文字
- [ ] **B3.2** `resumeFormatter.js` 按关键词分段
  - 识别：姓名、手机、邮箱、教育、工作、项目、技能
  - 解析不完美时保留 `rawText`，前端降级展示原文
- [ ] **B3.3** 异常处理：加密 PDF、纯图片扫描件 → 返回明确提示

### 阶段 B4：API 接口

#### 公开接口（面试官访问，无需登录）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/resume` | 获取当前简历结构化数据（展示页用） |
| GET | `/api/resume/file` | 获取原 PDF 文件流（预览/下载用） |

#### 管理接口（仅你自己，需鉴权）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/admin/upload` | 上传 PDF，覆盖旧简历并重新解析 |
| GET | `/api/admin/status` | 查看当前简历状态（更新时间、文件名） |
| POST | `/api/admin/login` | 简单密码登录，返回 Token |

- [ ] **B4.1** 实现公开接口（展示页、预览页依赖这两个）
- [ ] **B4.2** 实现管理接口 + 简单鉴权（Header 带 `Authorization: Bearer xxx`）
- [ ] **B4.3** 统一响应格式 `{ code, message, data }`

### 阶段 B5：部署到你的域名

- [ ] **B5.1** 买/已有域名，解析 A 记录到云服务器 IP
- [ ] **B5.2** 服务器安装 Node.js + Nginx + PM2
- [ ] **B5.3** Nginx 配置示例
  ```nginx
  server {
      listen 443 ssl;
      server_name 你的域名.com;

      # 前端 H5 静态文件
      location / {
          root /var/www/ccyuniapp/h5;
          try_files $uri $uri/ /index.html;
      }

      # 后端 API
      location /api/ {
          proxy_pass http://127.0.0.1:3000;
      }
  }
  ```
- [ ] **B5.4** 配置 HTTPS（Let's Encrypt 免费证书）
- [ ] **B5.5** PM2 守护后端：`pm2 start src/app.js --name resume-api`

## 五、面试当天怎么用

```
1. 面试官：「带简历了吗？」
2. 你：「有的，在线版方便一点」→ 发链接 https://你的域名.com
3. 对方手机打开 → 看到排版好的简历
4. 对方说「有 PDF 版吗？」→ 点页面底部「查看原版 PDF」
5. 面试结束，无需收回任何文件
```

**链接永久有效**，你改 PDF 重新上传后，同一个链接内容自动更新。

## 八、注意事项

1. **解析不完美没关系**：规则解析可能分段不准，前端始终有 `rawText` 降级展示；后续可手动微调 JSON 或加 AI 解析。
2. **管理页要藏好**：`/admin` 加密码，别在公开页放入口链接。
3. **域名要稳**：面试前提前测一遍链接，确保手机 4G/5G 也能秒开。
4. **PDF 扫描件**：如果是图片型 PDF，文字提取会失败，建议用 Word 导出的文字版 PDF。
5. **隐私**：简历里有手机号，公开页可以考虑只展示邮箱、面试时再口头报手机（按需调整）。
